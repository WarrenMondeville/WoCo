# ET 项目登录流程分析

> 本文档详细分析 WoCo 项目基于 ET 框架的登录流程，包括客户端 UI、NetClient、Realm、Gate 各层的职责与消息流转。

---

## 一、流程概览

项目采用经典的 **Realm → Gate** 双层登录架构：

```
[UI 登录界面] → [ClientSenderComponent] → [NetClient Fiber] → [Router] → [Realm] → [Gate] → [Session 建立] → [LoginFinish 事件]
```

---

## 二、各阶段详解

### 2.1 客户端 UI 层

**入口文件：** `UILoginComponentSystem.cs` / `UILSLoginComponentSystem.cs`

- 用户输入账号密码，点击登录按钮
- 调用 `LoginHelper.Login(root, account, password)`（`.Coroutine()` 异步执行）

```csharp
// UILoginComponentSystem.OnLogin
public static void OnLogin(this UILoginComponent self)
{
    LoginHelper.Login(
        self.Root(), 
        self.account.GetComponent<InputField>().text, 
        self.password.GetComponent<InputField>().text).Coroutine();
}
```

> Demo 模式使用 `UILogin`，LockStep 模式使用 `UILSLogin`，两者都调用同一套 `LoginHelper.Login`。

---

### 2.2 LoginHelper 与 ClientSenderComponent

**入口文件：** `LoginHelper.cs`

- 移除旧的 `ClientSenderComponent`（用于断线重连场景）
- 创建新的 `ClientSenderComponent`
- 调用 `LoginAsync` 完成登录，获得 `PlayerId`
- 设置 `PlayerComponent.MyId`
- 发布 `LoginFinish` 事件（如创建大厅 UI）

```csharp
// LoginHelper.Login
public static async ETTask Login(Scene root, string account, string password)
{
    root.RemoveComponent<ClientSenderComponent>();
    ClientSenderComponent clientSenderComponent = root.AddComponent<ClientSenderComponent>();
    long playerId = await clientSenderComponent.LoginAsync(account, password);
    root.GetComponent<PlayerComponent>().MyId = playerId;
    await EventSystem.Instance.PublishAsync(root, new LoginFinish());
}
```

---

### 2.3 ClientSenderComponent.LoginAsync（进程内 RPC）

**入口文件：** `ClientSenderComponentSystem.cs`

- 创建 NetClient 专用的 Fiber（`SceneType.NetClient`）
- 通过 `ProcessInnerSender` 向 NetClient Fiber 发送 `Main2NetClient_Login` 消息
- 等待 `NetClient2Main_Login` 响应，获取 `PlayerId` 并返回

```csharp
// ClientSenderComponentSystem.LoginAsync
public static async ETTask<long> LoginAsync(this ClientSenderComponent self, string account, string password)
{
    self.fiberId = await FiberManager.Instance.Create(SchedulerType.ThreadPool, 0, SceneType.NetClient, "");
    self.netClientActorId = new ActorId(self.Fiber().Process, self.fiberId);
    
    Main2NetClient_Login main2NetClientLogin = Main2NetClient_Login.Create();
    main2NetClientLogin.OwnerFiberId = self.Fiber().Id;
    main2NetClientLogin.Account = account;
    main2NetClientLogin.Password = password;
    NetClient2Main_Login response = await self.Root().GetComponent<ProcessInnerSender>()
        .Call(self.netClientActorId, main2NetClientLogin) as NetClient2Main_Login;
    return response.PlayerId;
}
```

### 2.4 NetClient：Main2NetClient_LoginHandler（核心网络登录）

**入口文件：** `Main2NetClient_LoginHandler.cs`

| 步骤 | 操作 |
|------|------|
| ① | 初始化 `RouterAddressComponent`，通过 HTTP 访问 `ConstValue.RouterHttpHost:RouterHttpPort`（默认 127.0.0.1:30300）获取路由与 Realm 地址 |
| ② | 调用 `RouterAddressComponent.GetRealmAddress(account)`，根据账号哈希选择 Realm 地址 |
| ③ | 创建到 Realm 的 UDP Router Session，发送 `C2R_Login`（account、password） |
| ④ | 接收 `R2C_Login`，内含 Gate 的 `Address`、`Key`、`GateId` |
| ⑤ | 使用返回的 Address 创建到 Gate 的 Session，并保存到 `SessionComponent` |
| ⑥ | 向 Gate 发送 `C2G_LoginGate`（Key、GateId） |
| ⑦ | 接收 `G2C_LoginGate`，得到 `PlayerId`，写回 `response.PlayerId` |

```csharp
// Main2NetClient_LoginHandler 核心流程
RouterAddressComponent routerAddressComponent =
    root.AddComponent<RouterAddressComponent, string, int>(ConstValue.RouterHttpHost, ConstValue.RouterHttpPort);
await routerAddressComponent.Init();

IPEndPoint realmAddress = routerAddressComponent.GetRealmAddress(account);

using (Session session = await netComponent.CreateRouterSession(realmAddress, account, password))
{
    C2R_Login c2RLogin = C2R_Login.Create();
    c2RLogin.Account = account;
    c2RLogin.Password = password;
    r2CLogin = (R2C_Login)await session.Call(c2RLogin);
}

Session gateSession = await netComponent.CreateRouterSession(
    NetworkHelper.ToIPEndPoint(r2CLogin.Address), account, password);
root.AddComponent<SessionComponent>().Session = gateSession;

C2G_LoginGate c2GLoginGate = C2G_LoginGate.Create();
c2GLoginGate.Key = r2CLogin.Key;
c2GLoginGate.GateId = r2CLogin.GateId;
G2C_LoginGate g2CLoginGate = (G2C_LoginGate)await gateSession.Call(c2GLoginGate);

response.PlayerId = g2CLoginGate.PlayerId;
```

### 2.5 服务端 Realm：C2R_LoginHandler

**入口文件：** `C2R_LoginHandler.cs`

| 步骤 | 操作 |
|------|------|
| ① | 根据 `account` 哈希选择 Gate：`RealmGateAddressHelper.GetGate(zone, account)` |
| ② | 向 Gate 发起内部 RPC `R2G_GetLoginKey`，获取临时 Key |
| ③ | 将 Gate 的 `Address`、`Key`、`GateId` 填入 `R2C_Login` 返回客户端 |
| ④ | 约 1 秒后关闭 Realm Session |

```csharp
// C2R_LoginHandler
StartSceneConfig config = RealmGateAddressHelper.GetGate(session.Zone(), request.Account);

R2G_GetLoginKey r2GGetLoginKey = R2G_GetLoginKey.Create();
r2GGetLoginKey.Account = request.Account;
G2R_GetLoginKey g2RGetLoginKey = (G2R_GetLoginKey) await session.Fiber().Root.GetComponent<MessageSender>()
    .Call(config.ActorId, r2GGetLoginKey);

response.Address = config.InnerIPPort.ToString();
response.Key = g2RGetLoginKey.Key;
response.GateId = g2RGetLoginKey.GateId;
```

---

### 2.6 服务端 Gate：R2G_GetLoginKeyHandler

**入口文件：** `R2G_GetLoginKeyHandler.cs`

- 生成随机 64 位 Key
- 在 `GateSessionKeyComponent` 中存储 `key → account` 映射
- 将 Key 和 GateId 返回给 Realm

```csharp
// R2G_GetLoginKeyHandler
long key = RandomGenerator.RandInt64();
scene.GetComponent<GateSessionKeyComponent>().Add(key, request.Account);
response.Key = key;
response.GateId = scene.Id;
```

---

### 2.7 服务端 Gate：C2G_LoginGateHandler

**入口文件：** `C2G_LoginGateHandler.cs`

| 步骤 | 操作 |
|------|------|
| ① | 用 Key 在 `GateSessionKeyComponent` 中查找 account，失败则返回 `ERR_ConnectGateKeyError` |
| ② | 根据 account 查找或创建 `Player` |
| ③ | 新玩家：创建 `PlayerSessionComponent`、注册到 `LocationType.GateSession`、`LocationType.Player` |
| ④ | 老玩家重连：若在 Room 内则调用 `CheckRoom` 处理重连；否则直接更新 Session |
| ⑤ | 返回 `PlayerId` |

```csharp
// C2G_LoginGateHandler 关键逻辑
string account = root.GetComponent<GateSessionKeyComponent>().Get(request.Key);
if (account == null)
{
    response.Error = ErrorCore.ERR_ConnectGateKeyError;
    response.Message = "Gate key验证失败!";
    return;
}

Player player = playerComponent.GetByAccount(account);
if (player == null)
{
    player = playerComponent.AddChild<Player, string>(account);
    // ... 创建 PlayerSessionComponent、注册位置 ...
}
else
{
    // 断线重连逻辑：检查是否在战斗中，或在 lobby
}

response.PlayerId = player.Id;
```

## 三、消息协议一览

| 消息 | 方向 | 说明 |
|------|------|------|
| `C2R_Login` | Client → Realm | 携带 Account、Password 请求登录 |
| `R2C_Login` | Realm → Client | 返回 Gate 的 Address、Key、GateId |
| `R2G_GetLoginKey` | Realm → Gate | 内部 RPC，请求登录 Key |
| `G2R_GetLoginKey` | Gate → Realm | 返回 Key、GateId |
| `C2G_LoginGate` | Client → Gate | 携带 Key、GateId 完成 Gate 认证 |
| `G2C_LoginGate` | Gate → Client | 返回 PlayerId（登录成功） |

---

## 四、设计要点

1. **Realm 职责单一**：仅负责分配 Gate、获取 Key，不校验账号密码（当前实现无密码校验逻辑）。
2. **Key 一次性**：Realm 向 Gate 申请 Key，客户端持 Key 连接 Gate，Gate 验证后建立 Session。
3. **账号哈希选服**：通过 `account.GetLongHashCode()` 选择 Gate 和 Realm，实现同一账号固定进入同一 Gate。
4. **双 UI 入口**：Demo 的 `UILogin` 与 LockStep 的 `UILSLogin` 共用同一套 `LoginHelper.Login`，登录完成后分别监听 `LoginFinish` 创建 `UILobby` 或 `UILSLobby`。
